# 架构

*[English](../architecture.md) · [中文]*

Interview Trainer 把可复用的**引擎**和你的个人**数据**分开。理解这三层，就明白它为什么能跨岗位用、
为什么在漫长的求职季里还能保持准确。

## 三层结构

```
┌─────────────────────────────────────────────────────────────┐
│ 引擎层（本插件 — 共享、领域无关）                              │
│   skills/  — 一个 /interview 路由器 + 9 个 skill              │
│   无个人数据。无硬编码公司。无固定话题。                       │
└───────────────┬─────────────────────────────────────────────┘
                │ 读 ↑↓ 写
┌───────────────┴───────────────┐   ┌─────────────────────────┐
│ 配置层（你的）                  │   │ 数据层（自动维护）        │
│  profile.md                    │   │  data/pipeline.md       │
│  knowledge-base/               │   │  data/capability-...    │
│   话题、类比、STAR、图、         │   │  data/records/          │
│   方法论、公司风格、答题规范      │   │  data/mock-records/     │
│  （一次填好）                   │   │  data/prep/             │
└────────────────────────────────┘   └─────────────────────────┘
```

- **引擎层**随插件发布，永远不含你的数据，运行时从你的工作区按需读取。
- **配置层**是你拥有的知识：你是谁、锁定的量化数据、话题体系、类比和 STAR 故事、你关心的公司风格。
  用 `/interview setup` 一次性建好，之后逐步扩充。
- **数据层**是引擎替你写的状态：管线漏斗、能力画像、每次模拟和真实面试记录。基本不用手动编辑。

你的**工作区**就是你启动 Claude Code 的那个目录。没有全局配置——引擎相对当前目录解析
`profile.md`、`knowledge-base/`、`data/`。

## 9 个 skill

| Skill | 角色 |
|-------|------|
| `interview` | 路由器 + 智能导航 + 状态仪表盘 + 分析 + 投递编排 |
| `interview-setup` | 一次性引导：建配置 + 数据骨架 |
| `mock-interview` | 扮演面试官；一问一答；评分；跑复习回路 |
| `interview-debrief` | 面试后回忆 + 教练诊断；写记录、更新画像和管线 |
| `deep-review` | L1-L5 渐进闯关 + 跨会话掌握度档案 |
| `review-skill` | 技能点一站式复习卡片 |
| `interview-coach` | 生成结构化高质量回答 |
| `interview-prep` | 公司 + 轮次针对性准备计划 |
| `resume-tailor` | JD 匹配评分 + ATS 优化 + 不编造重写 |

## 反馈回路（为什么越用越强）

```
真实面试 ──/debrief──► 薄弱点写入能力画像
                            │
                            ├──► /deep-review 把它当作补强关
                            ├──► /mock-interview --weak 优先练它
                            └──► /interview 导航推荐你先补它

模拟面试 ──► 更新能力画像（话题水平、准备度）
深度复习 ──► 更新掌握度档案 + <3 分倒灌成薄弱点
所有数据 ──► /interview 读管线 + 画像 + 记录，推荐下一步
```

周二面试暴露的薄弱点，周三自动变成针对性闯关，并一直出现在导航推荐里，直到你补上。

## 数据一致性纪律

状态文件会漂移（改了一个忘了另一个）。引擎按固定优先级认定事实——真实记录 > 管线 > 能力画像 > 文档——
并从当前表格**重算**漏斗和准备度，而不信任旧的统计块。发现漂移会输出**数据告警**，
而不是静默"修正"你的历史。这条规则写在 `knowledge-base/answer-norms.md`。

## 语言

skill 指令用中文写（经过实战打磨的 proven prompt），但每个 skill 都被要求**用你的语言主持对话**。
你用英文，整套就跑英文；中文对你不可见。
