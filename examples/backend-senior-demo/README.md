# Example Workspace — "backend-senior-demo"

> ⚠️ **Everything here is FICTIONAL.** Candidate *Jordan Park*, company *Meridian* (a made-up
> payments platform), projects, and metrics are invented to show the **shape** of a backend
> engineer's workspace. Copy the structure, not the content.

Shows how the same engine adapts to a **backend** role — the topic taxonomy, company style, and
STAR stories are all backend-flavored (concurrency, databases, distributed systems), while the
engine (mock/debrief/drill/profile) is unchanged.

```
backend-senior-demo/
├── profile.md
├── knowledge-base/
│   ├── topics.md              # backend taxonomy (concurrency, DB, distributed, API…)
│   ├── star-stories.md
│   └── company-styles/Meridian.md
└── data/
    ├── pipeline.md
    ├── capability-profile.md
    └── mock-records/2026-02-03-system-design-mock.md
```

Try it: `cp -r examples/backend-senior-demo ~/be-demo && cd ~/be-demo`, then `/interview status`.
