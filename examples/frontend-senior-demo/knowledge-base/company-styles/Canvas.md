# Canvas Interview Style (FICTIONAL DEMO)

> Inherits archetype: **system-design** (primary) + **behavioral** (secondary)

## Per-round focus
- Round 1: JS/framework fundamentals + a UI coding task
- Round 2: performance deep-dive + rendering internals
- Round 3: frontend system design (real-time / collaborative)
- Round 4 / hiring manager: product sense + collaboration

## Follow-up style
- Product-aware: "how did this affect the user / conversion?"
- Perf-obsessed: "field data or lab? p75? what's on the critical path?"
- Pushes rendering internals: "when does React bail out of a re-render?"

## Common questions
- Reconciliation, keys, memoization trade-offs
- LCP/INP diagnosis and fixes; bundle strategy
- Design a collaborative canvas / a data-heavy table / an offline-capable editor
- A11y: keyboard nav and screen-reader support

## Core traits
1. Perf + product tied together; numbers must be field-grounded
2. Cares about accessibility as a first-class requirement
3. Values taste in component/API design
