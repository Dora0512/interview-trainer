# Archetype：稳定性型 / Stability-Focused

> 典型代表：对线上质量要求极高、看重工程成熟度的公司（金融、交易、出行、旅游等）。

## 各轮侧重
- 一面：基础 + 性能优化入门 + 项目经历
- 二面：**稳定性治理 + 线上问题排查 + 监控体系 + 架构**
- 三面/总监面：技术全局观 + 团队管理 + 业务理解

## 追问风格
- 比算法深挖型温和，给思考时间，但深度追问不含糊
- 不只问"你做过什么优化"，更问"**线上出了问题你怎么排查**"
- 排查链条追问模板：
  ```
  "你们 crash 率多少？" → "这个数字合理吗？行业水平多少？"
  → "怎么采集的？用什么工具？" → "Top 3 类型是什么？"
  → "举一个你排查过的具体故障，从发现到修复的完整链路"
  → "修完怎么验证？灰度流程是什么？"
  ```

## 高频题型
- crash/ANR/卡顿治理、线上事故复盘
- 监控体系（采集→上报→聚合→告警→看板）
- 高流量/大促保障、灰度与回滚机制

## 核心特征
1. 稳定性第一，重"监控→定位→修复→验证"闭环
2. 看重工程成熟度：监控、灰度、复盘、代码规范
3. 喜欢结合真实业务场景出题
