# Archetype：系统设计型 / System-Design-Heavy

> 典型代表：高级/专家岗，重点考查从零设计系统的能力和架构权衡。

## 各轮侧重
- 一面：基础 + 项目，确认技术底子
- 二面：架构能力 + 一道系统设计题
- 三面/终面：**开放式系统设计 + 技术决策 + 演进规划**

## 追问风格
- 给一个开放问题，看你怎么拆解、分层、做取舍
- 引导式："可以先说分几层""如果 QPS 翻 10 倍呢？""这里的瓶颈在哪？"
- 关注边界和极端情况："如果某个依赖挂了怎么办？"
- 关注演进："第一版怎么做，规模上来后怎么改？"

## 高频题型
- 设计一个高性能列表/Feed 系统
- 设计一个监控/限流/推送/缓存系统
- 设计一个 <你领域> 的核心服务，要求可扩展、高可用

## 核心特征
1. 没有标准答案，看思路和权衡
2. 重分层、解耦、扩展性、容错
3. 期待你主动说出 trade-off 和演进路径

## 答题骨架
需求澄清 → 容量估算 → 整体分层 → 核心模块设计 → 数据流 → 瓶颈与扩展 → 容错与降级 → 演进
