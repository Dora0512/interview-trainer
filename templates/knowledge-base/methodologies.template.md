# 方法论库 / Methodologies

> 一组可复用的「答题方法论」。模拟面试和深度复习评分「表达节奏」「数据驱动」「叙事」
> 等维度时会引用它们。下面是 3 个通用方法，可直接用，也可加你自己的。

---

## 方法一：答题三步法（答到重点）/ Answer-to-the-Point

**何时用**：所有题目，尤其是容易"说不到重点 / 答偏"的时候。

**步骤**：
1. **停 2 秒** — 别急着开口。先在心里确认面试官到底在问什么。
2. **翻译考点** — 把问题翻译成"他想验证我哪个能力"（如问"冷启动慢怎么办"，考点其实是"你有没有系统的性能优化方法论"）。
3. **锚点先行** — 先给结论/框架（"主要从三个方向：A、B、C"），再展开。别一上来就钻细节。

**反例**：被问一个开放问题，立刻跳进某个技术细节，讲了 3 分钟没回到面试官真正想听的点。

---

## 方法二：量化数据衡量五步法 / Quantified-Result Framework

**何时用**：任何一处提到量化成果时（"我把 X 优化了 N%"），尤其被追问"这个数字怎么来的"。

**步骤**：
1. **指标** — 明确是什么指标（不是"性能变好了"，而是"P50 延迟"）。
2. **测量** — 怎么测的、用什么工具、统计口径是什么。
3. **排除** — 怎么排除其他变量的干扰（同期没有别的改动？流量结构没变？）。
4. **归因** — 把效果拆解到你具体做的动作上（哪一步贡献了多少）。
5. **业务连接** — 这个技术指标怎么连到业务价值（延迟降低 → 转化提升 → 收入）。

**反例**："我优化了启动速度"——没有指标、没有测量、没有归因，面试官一追问就崩。

---

## 方法三：问题侦查时间线叙事 / Investigation Timeline Narrative

**何时用**：性能/稳定性/线上故障类话题——讲"你怎么排查并解决一个问题"。

**框架（T0-T5 六段式）**：
- **T0 现象** — 什么时候、什么场景、什么指标异常（带数字）
- **T1 止血** — 第一时间怎么止损（回滚/降级/限流）
- **T2 定位** — 用什么工具、怎么缩小范围、关键证据是什么
- **T3 根因** — 真正的根因（不是表象）
- **T4 修复** — 怎么改的、为什么这么改
- **T5 验证 + 复盘** — 怎么验证修好了、灰度流程、长效防护（监控/告警/规范）

**精简版（3+1 四段式）**：现象 → 定位 → 修复 → 防护。

**反例**：被问"讲一个你排查过的线上问题"，只说"是个内存泄漏，改了就好了"——没有时间线、没有侦查过程，体现不出能力。
